§ 01

What Is the Teensy 4.1

The Teensy 4.1 is a microcontroller development board built around NXP's ARM Cortex-M7 chip — the same family of processors found in automotive ECUs and industrial controllers. It's programmed through the Arduino IDE (so the learning curve is gentle), but under the hood it's dramatically more powerful than a standard Arduino.

For YoshiOS, the Teensy is your physical I/O bridge: it reads every switch, knob, encoder, and button on the center console, then sends structured messages over USB serial to the Kingdel, where Electron picks them up and translates them into dashboard actions.

Why not just use an Arduino?

An Arduino Uno runs at 16 MHz with 2 KB of RAM. The Teensy 4.1 runs at 600 MHz with 1 MB of RAM — roughly 37× the clock speed. More importantly, it has native USB, meaning it shows up as a real USB device to the Kingdel with no FTDI chip in between. This makes serial communication faster, more reliable, and simpler to code on the Electron side.

3.3V Logic — Keep This in Mind

The Teensy's GPIO pins are 3.3V and not 5V-tolerant. Your Philmore switches and rotary encoders will wire to 3.3V (the 3.3V output pin on the board), not 5V. Feeding 5V into a Teensy pin can damage it permanently. The onboard VIN pin does accept 5V from USB, but that's for power — the signal pins are 3.3V only.

§ 02

Board Anatomy — Visual Walkthrough

The Teensy 4.1 is a long, narrow board designed to straddle a breadboard. Here's a simplified map of what lives where:

Fig. 01 Teensy 4.1 top-down layout. Green = digital GPIO, blue = analog inputs (also usable as digital), red = power/ground. The USB-C end faces the Kingdel.

Key Areas to Know

USB-C port (left end) — This is your only physical connection to the Kingdel. It supplies 5V power to the Teensy and carries all serial data. One cable does everything.

Program button (PGM) — If the Teensy ever gets stuck and won't accept new code, press this button while plugging in USB. It forces the board into bootloader mode. You'll rarely need it, but good to know it's there.

Pin 13 LED — An onboard orange LED wired to digital pin 13. This is the classic "blink" test target — if it blinks, your code is running.

GPIO pins (both edges) — The two long rows of pins along the board edges are where you'll wire your switches, encoders, and knobs. Each pin can be individually configured as an input (reading a switch state) or output (driving an LED). The analog pins (blue) can also read variable voltages — useful for potentiometers.

§ 04

Environment Setup

The Teensy uses the Arduino IDE for writing and uploading code (called "sketches"). But it needs a special add-on called Teensyduino that teaches the Arduino IDE how to talk to the Teensy hardware.

1

Install Arduino IDE

Download from arduino.cc/en/software. Version 2.x works. Install it normally — you won't write "Arduino code" per se, but the IDE is the editor + compiler + uploader all in one.

2

Install Teensyduino

Go to pjrc.com/teensy/td_download.html and grab the Teensyduino installer for your OS. Run it and point it at your Arduino IDE installation. This installs the Teensy board definitions, drivers, and libraries.

3

Select the Board

In Arduino IDE: Tools → Board → Teensyduino → Teensy 4.1. Also set Tools → USB Type → Serial (this is the default and what YoshiOS will use).

4

Plug In via USB-C

Connect the Teensy to your computer. It should show up under Tools → Port as something like COM3 (Windows) or /dev/ttyACM0 (Linux/Mac). If nothing appears, try a different USB cable — some cables are charge-only with no data lines.

Teensy Loader

When you install Teensyduino, it also installs a small "Teensy Loader" app that pops up during uploads. This is normal — it's the flash utility that pushes compiled code onto the board. It'll auto-launch when you click Upload in the Arduino IDE. Just let it do its thing.

§ 05

First Sketch — Blink + Serial Hello

Every microcontroller journey starts with blink. But let's level it up: we'll blink the onboard LED and send a serial message to the computer at the same time. This proves both halves of the YoshiOS bridge — hardware control and USB communication.

Arduino / Teensy
// YoshiOS Teensy Primer — Sketch 01
// Blink pin 13 LED + send serial messages

const int LED_PIN = 13;
unsigned long lastBlink = 0;
bool ledState = false;

void setup() {
  pinMode(LED_PIN, OUTPUT);
  Serial.begin(115200);  // USB serial speed

  // Wait up to 3 seconds for serial monitor to open
  while (!Serial && millis() < 3000) {}

  Serial.println("{\"src\":\"teensy\",\"event\":\"boot\"}");
}

void loop() {
  unsigned long now = millis();

  // Blink every 500ms
  if (now - lastBlink >= 500) {
    lastBlink = now;
    ledState = !ledState;
    digitalWrite(LED_PIN, ledState);

    // Send a heartbeat JSON line
    Serial.print("{\"src\":\"heartbeat\",\"ms\":");
    Serial.print(now);
    Serial.println("}");
  }
}
▶

Upload & Test

Click the Upload button (→ arrow) in the Arduino IDE. The Teensy Loader will flash the code. The onboard LED should start blinking. Open Tools → Serial Monitor (set baud to 115200) and you'll see JSON lines streaming in — that's exactly what Electron will eventually read.

Why JSON Lines?

We're sending one JSON object per line (newline-delimited JSON, aka "JSONL"). This is the simplest protocol for the Electron side to parse: read until \n, call JSON.parse(), done. No binary protocols, no escape sequences. Each message is self-describing with a "src" field so Electron knows what generated it.

§ 06

Wiring Your First Input — Rotary Encoder

A rotary encoder is the knob that clicks as you turn it — perfect for volume, menu navigation, or cycling through dashboard screens. It has two signal pins (A and B) that produce a pattern telling you which direction it was turned, plus a push-button pin for click-to-select.

Fig. 03 Rotary encoder wiring. Three signal wires + ground. The Teensy's internal pull-up resistors save you from needing external ones.
Encoder PinTeensy PinPurpose
GNDGNDCommon ground reference
+ (VCC)3.3VPower for the encoder module (if it has one — bare encoders don't need this)
CLK (A)Pin 2Rotation signal A — generates interrupts
DT (B)Pin 3Rotation signal B — direction detection
SWPin 4Push button — LOW when pressed
Arduino / Teensy
// YoshiOS Teensy Primer — Sketch 02
// Rotary encoder with button → JSON serial

const int PIN_A   = 2;   // CLK
const int PIN_B   = 3;   // DT
const int PIN_BTN = 4;   // SW (button)
const int LED_PIN = 13;

volatile int encoderPos = 0;
int lastReportedPos = 0;
bool lastBtn = HIGH;

void encoderISR() {
  // Read B to determine direction
  if (digitalRead(PIN_B) == LOW) {
    encoderPos++;   // clockwise
  } else {
    encoderPos--;   // counter-clockwise
  }
}

void setup() {
  pinMode(PIN_A, INPUT_PULLUP);
  pinMode(PIN_B, INPUT_PULLUP);
  pinMode(PIN_BTN, INPUT_PULLUP);
  pinMode(LED_PIN, OUTPUT);

  // Fire ISR on falling edge of A
  attachInterrupt(digitalPinToInterrupt(PIN_A),
                  encoderISR, FALLING);

  Serial.begin(115200);
  while (!Serial && millis() < 3000) {}
  Serial.println("{\"src\":\"teensy\",\"event\":\"boot\"}");
}

void loop() {
  // Report encoder changes
  int pos = encoderPos;
  if (pos != lastReportedPos) {
    int delta = pos - lastReportedPos;
    lastReportedPos = pos;

    Serial.print("{\"src\":\"encoder\",\"pos\":");
    Serial.print(pos);
    Serial.print(",\"delta\":");
    Serial.print(delta);
    Serial.println("}");
  }

  // Report button press (debounced)
  bool btn = digitalRead(PIN_BTN);
  if (btn != lastBtn) {
    delay(20);  // simple debounce
    btn = digitalRead(PIN_BTN);
    if (btn != lastBtn) {
      lastBtn = btn;
      if (btn == LOW) {
        Serial.println("{\"src\":\"encoder_btn\",\"val\":1}");
        digitalWrite(LED_PIN, HIGH);
      } else {
        Serial.println("{\"src\":\"encoder_btn\",\"val\":0}");
        digitalWrite(LED_PIN, LOW);
      }
    }
  }
}

An interrupt is a hardware signal that pauses your main loop() code, runs a small function (the ISR — interrupt service routine), and then resumes where it left off. We use one here because the encoder spins fast — if we only checked it inside loop(), we might miss pulses. The interrupt guarantees we catch every single click of the encoder, even if loop() is busy doing something else.

On the Teensy 4.1, every digital pin supports interrupts — a big advantage over basic Arduinos where only a few pins support them. This means you can attach interrupts to as many encoders and buttons as you need for the full center console.

Mechanical switches don't make clean contact — when you press a button, the metal contacts "bounce" for a few milliseconds, rapidly toggling between connected and disconnected. Without debouncing, one button press might register as 5–10 presses. The simple delay(20) approach above works fine for buttons. For the full console, you'll eventually want a library like Bounce2 that handles this more elegantly without blocking the main loop.

§ 07

Talking to Electron — USB Serial Bridge

Once the Teensy is sending JSON lines over USB, the Kingdel side needs to listen. In YoshiOS, this happens in the Electron main process using the serialport npm package. Here's the basic bridge:

Node.js / Electron Main
// In your Electron main process (main.js)

const { SerialPort } = require('serialport');
const { ReadlineParser } = require('@serialport/parser-readline');

// Find the Teensy — it shows up as USB Serial
async function connectTeensy() {
  const ports = await SerialPort.list();
  const teensy = ports.find(p =>
    p.manufacturer?.includes('Teensyduino') ||
    p.vendorId === '16C0'
  );

  if (!teensy) {
    console.log('Teensy not found — running without controls');
    return null;
  }

  const port = new SerialPort({
    path: teensy.path,
    baudRate: 115200
  });

  const parser = port.pipe(
    new ReadlineParser({ delimiter: '\n' })
  );

  parser.on('data', (line) => {
    try {
      const msg = JSON.parse(line);
      console.log('Teensy:', msg);

      // Forward to the renderer via IPC
      mainWindow.webContents.send(
        'teensy-event', msg
      );
    } catch (e) {
      // Skip malformed lines
    }
  });

  port.on('error', (err) => {
    console.error('Serial error:', err);
  });

  return port;
}
React / Renderer
// In your React renderer (e.g., Dashboard.jsx)

const { ipcRenderer } = require('electron');

ipcRenderer.on('teensy-event', (event, msg) => {
  switch (msg.src) {
    case 'encoder':
      // msg.delta is +1 (CW) or -1 (CCW)
      handleVolumeChange(msg.delta);
      break;

    case 'encoder_btn':
      if (msg.val === 1) handleEncoderPress();
      break;

    case 'toggle_ac':
      setAcEnabled(msg.val === 1);
      break;

    case 'heartbeat':
      // Teensy is alive — update status LED
      setTeensyConnected(true);
      break;
  }
});
Install the serial library

In your YoshiOS project directory, run: npm install serialport @serialport/parser-readline. The serialport package has native bindings, so you may need build tools installed on the Kingdel (Python, C++ compiler). On Windows this usually means installing the "Desktop development with C++" workload from Visual Studio Build Tools.

Suggested Learning Path

①

Get blink running

Sketch 01 from § 05. Confirms your toolchain works end to end.

②

Wire one encoder, run Sketch 02

Watch JSON lines flow in the Serial Monitor. Twist the knob, press the button, see the messages.

③

Add the serialport bridge to YoshiOS

The Node.js code from § 07. Once this works, encoder twists should trigger events inside your Electron app.

④

Add one toggle switch

Same pattern as the encoder button: wire to a digital pin, INPUT_PULLUP, debounce, send JSON. The Philmore 30-370 is a good first one.

⑤

Scale to the full console

Add controls one at a time. Keep the JSON protocol consistent. The Teensy has 55 digital pins — more than enough for the full center console layout.

Useful Libraries for Later

Encoder.h — PJRC's own library for high-performance rotary encoder reading. Handles quadrature decoding better than our manual ISR approach. Install via Arduino Library Manager.

Bounce2 — Robust debouncing for all your buttons and toggles. Cleaner than manual delay-based debouncing.

ArduinoJson — If your JSON messages get complex, this library builds JSON properly without string concatenation. Overkill for simple messages but nice for larger payloads.
